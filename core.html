mvacfg.core.mva_study(signame, sigsmp, bkgname, bkgsmp, cfg, outdir='mva_outputs', is_sig='is_sig', raise_if_matches=False, return_dir=False, return_cid=False)[source]¶

Main function to perform a MVA study. The results are stored in three different files: one storing the histograms and the ROC curve, another with the configuration used to run this function, and the last stores the proper class to store the MVA algorithm.

Parameters:
  • signame (str) – signal sample name.
  • sigsmp (pandas.DataFrame) – signal sample.
  • bkgname (str) – background sample name.
  • bkgsmp (pandas.DataFrame) – background sample.
  • cfg (ConfMgr or dict) – configurable for the MVA manager.
  • outdir (str) – output directory. By default is set to “mva_outputs”. The full output directory is actually determined from the configuration ID of the study so, assuming the default value, it would be under “mva_outputs/mva_<configuration ID>”.
  • is_sig (str) – name for the additional column holding the signal condition.
  • raise_if_matches (bool) – if set to True, a LookupError will be raised if it is found a configuration matching the input. This is useful when running many configurations. For example, if one wants to skip those which have already been studied.
  • return_dir (bool) – if set to True, the directory where the outputs are saved is also returned.
  • return_cid (bool) – if set to True, return also the configuration ID.
Returns:

MVA manager, training and testing samples it might also return the directory where the outputs are saved and the configuration ID.
Return type:

tuple(MVAmgr, pandas.DataFrame, pandas.DataFrame (, str) (, int))
Raises:

LookupError – if “raise_if_matches = True”, and a configuration matching the input is found.
